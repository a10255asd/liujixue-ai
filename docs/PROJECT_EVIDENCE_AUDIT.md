# 项目证据审计

更新时间：2026-07-16

本文档记录 `content/projects.json` 中项目的真实交付成熟度。项目文案、训练路径引用和简历表达不能替代可运行代码与验收证据。

## 成熟度协议

| 状态 | 含义 | 必要证据 |
| --- | --- | --- |
| `blueprint` | 只有目标、架构与交付模板 | 不允许展示 Demo、测试通过或已交付表述 |
| `prototype` | 存在可运行的最小闭环 | Demo 路径、验证日期、至少 1 条命令和 2 项验证产物 |
| `verified` | 可独立部署并完成验收 | `prototype` 全部证据，加独立仓库地址与完整验收记录 |

`resumeBullet` 在 `verified` 前只能显示为目标表达，不得直接用于简历。

## Prompt 原型边界

入口：`/labs/prompt-regression`

- 只使用仓库内固定输入、期望和响应，不调用模型，也不需要 API Key。
- 同时计算 Schema 通过率与业务通过率，避免把“JSON 可解析”误当作“回答正确”。
- 延迟和成本是固定夹具字段，只验证报表与回归流程，不代表真实模型性能。
- 通过 `npm run test:unit` 和 `npm run build` 验证。

## RAG 原型边界

入口：`/labs/rag-retrieval`

- 使用仓库内 8 份确定性内部手册，按 16 个稳定章节边界摄取，不连接外部数据源。
- 关键词基线使用字符词面重叠；混合检索加入稀有词权重和显式概念标签，不伪装成真实向量相似度。
- 13 条固定评估题包含 12 条有答案问题和 1 条知识库外问题，输出 Hit@3、MRR、引用覆盖和未知问题拒答率。
- 回答直接抽取命中文本并附稳定文档、版本和章节引用，不调用生成模型。
- 通过 `npm run test:unit`、`npm run build` 和双端 Playwright 验证。

## 受控 Agent 原型边界

入口：`/labs/controlled-agent`

- 使用确定性状态机执行五类固定场景，不调用模型，不连接真实外部工具。
- 每次工具调用先检查权限和步数预算；高风险删除必须等待人工审批。
- 临时故障重试复用同一幂等键，只有成功结果能提交一次副作用。
- 越权场景在工具调用前失败；预算场景达到上限后不再继续循环。
- 事件轨迹包含连续序号、稳定时间、工具、尝试次数、幂等键和副作用状态。

## Agent 评测原型边界

入口：`/labs/agent-evaluation`

- V2 直接复用受控 Agent 的五类轨迹；V1 使用仓库内固定回归事实，不伪装成真实线上版本。
- 评测固定覆盖结果、轨迹、权限、预算和副作用，输出具体样本与首个回归位置。
- 门禁同时检查综合分、单样本最低分与关键安全违规；权限、预算或副作用失败不能被均分抵消。
- 当前不接模型评判、真实 Trace、延迟或成本采集，报告只证明确定性评测流程可以复算。
