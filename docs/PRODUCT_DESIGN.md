# AI 学习知识库子站设计文档 v1.0

更新时间：2026-07-15

## 1. 项目背景

用户正在准备投递 Agent 工程师岗位，但当前对 AI Agent 工程体系不熟。

这个子站不是单纯展示“我懂 AI”，而是把一个普通开发者从 0 学 AI、做项目、刷面试、准备求职的全过程产品化。它同时服务四件事：

- 自己学习：把零散资料沉淀成可执行路线。
- 求职展示：让面试官看到持续学习、工程拆解和项目实践能力。
- 长期内容资产：形成 `liujixue.cn` 体系下的 AI 子品牌。
- 后续产品化：未来可以扩展学习进度、错题本、模拟面试和 AI 辅导。

## 2. 子站定位

推荐域名：`ai.liujixue.cn`

推荐名称：

- 中文：刘鸡血 AI 学习库
- 英文：Jixue AI Lab
- 页面标题：从 0 开始学 AI Agent 工程

核心定位：

> 给 0 基础开发者，尤其是想转 AI Agent 工程师的人，用一条可执行路线学 AI、做项目、刷面试、准备求职。

更聚焦的差异化表达：

> 给后端/全栈开发者看的 AI Agent 工程师转型知识库。

## 3. 目标用户

### 3.1 第一用户：站长本人

- 已有 Java 后端、网站、小程序和服务器经验。
- 需要补齐 LLM、RAG、Agent、MCP、Eval、AI 应用工程化能力。
- 希望用站点内容反向服务简历、项目讲解和面试准备。

### 3.2 外部用户

- 会写业务代码，但不知道 AI 应用从哪里学。
- 会用 ChatGPT/Cursor，但不理解 API、RAG、Agent 的工程边界。
- 想准备 AI 工程、Agent 工程、LLM 应用开发相关面试。

## 4. 产品目标

第一阶段目标：

- 让用户知道 AI Agent 工程师要学什么。
- 让用户可以按路线学习，而不是只收藏资料。
- 让用户可以按模块刷 AI 面试题。
- 让用户看到每个知识点如何连接到项目实战和面试回答。
- 让后续 AI agent 可以按内容模型持续补题、补知识点、补项目。

非目标：

- 第一版不做登录系统。
- 第一版不做用户学习进度。
- 第一版不做 AI 自动答题评分。
- 第一版不做付费课程。
- 第一版不做泛资讯流。

## 6. 首页设计

首页不要做传统营销页，要像“学习控制台”。

首屏：

```text
H1：从 0 开始学 AI Agent 工程
副标题：面向开发者的 AI 学习路线、面试题库和项目实战知识库。

主按钮：
- 开始学习路线
- 刷 AI 面试题
- 看 Agent 项目实战
```

首页模块：

- 当前推荐路线：0 基础 -> LLM API -> RAG -> Agent -> MCP -> Eval -> 求职。
- 高频面试题：展示 6-8 道题。
- Agent 工程核心概念：工具调用、Agent Loop、记忆、规划、评估、可观测性。
- 项目实战：RAG 知识库、AI 面试官、简历优化 Agent、MCP 文件助手。
- 学习日志：展示最近学习记录，强化“真实从 0 学起”的人设。

## 7. 学习路线模块

页面：`/roadmap`

路线分 7 阶段。

### 阶段 1：AI 使用基础

目标：

- 理解 ChatGPT、Claude、Cursor、Codex 的角色。
- 理解模型、Token、上下文窗口、温度、系统提示词。

产出：

- 能解释一个 LLM 请求发生了什么。
- 能写出稳定的基础 Prompt。

### 阶段 2：LLM API 开发

目标：

- 调用 LLM API。
- 理解流式输出、结构化输出、工具调用、错误重试、成本控制。

产出：

- 做一个 Prompt 调试器。
- 做一个结构化 JSON 输出示例。

### 阶段 3：Prompt 与 Context Engineering

目标：

- 任务拆解。
- Few-shot 示例。
- 上下文压缩。
- 输出格式约束。
- 长任务交接。

产出：

- 做一个“需求转 AI 任务包”的 Prompt 模板。

### 阶段 4：RAG 知识库

目标：

- 理解 Embedding、Chunk、向量库、检索、重排、引用来源、幻觉控制。

产出：

- 做一个 AI 学习资料问答知识库。

### 阶段 5：Agent 工程

目标：

- 理解 Agent Loop、工具调用、状态、记忆、任务规划、人工确认、多 Agent。

产出：

- 做一个能读取资料、拆任务、输出执行计划的 Agent。

### 阶段 6：MCP 和工具生态

目标：

- 理解 MCP Server、Client、Tools、Resources、Prompts。
- 理解工具权限和安全边界。

产出：

- 做一个本地文件/知识库 MCP 助手。

### 阶段 7：工程化和求职

目标：

- 理解日志、Trace、Eval、监控、部署、回滚、安全。
- 准备简历、项目讲解和模拟面试。

产出：

- 一个可展示的 Agent 项目。
- 一套 AI 面试题回答稿。
- 一篇求职复盘日志。

## 9. Agent 专题模块

页面：`/agent`

专题页建议：

- `/agent/what-is-agent`：什么是 Agent。
- `/agent/tool-calling`：工具调用。
- `/agent/agent-loop`：Agent Loop。
- `/agent/state-memory`：状态和记忆。
- `/agent/planning`：任务规划。
- `/agent/multi-agent`：多 Agent。
- `/agent/guardrails`：安全边界。
- `/agent/observability`：可观测性。
- `/agent/evaluation`：Agent 评估。

每个专题必须回答：

- 它解决什么问题。
- 它和普通 LLM 调用有什么区别。
- 工程上怎么实现。
- 有哪些失败模式。
- 面试官会怎么问。

### 求职闭环

`/career` 不做泛化职业建议。它把常见 Agent 工程师 JD 拆成 8 个能力域，每个能力域必须关联知识点、面试题、项目和可验证证据。自测只按已勾选的确定性能力项计算，首个缺口决定推荐周次；8 周计划每周都必须有交付物和退出标准。

## 14. 视觉风格

关键词：

- 高端知识库
- 工程手册
- 清晰、克制、专业
- 不要花哨科技感
- 不要大面积紫蓝渐变
- 不要营销页套路

推荐风格：

- 背景：近白 / 暖灰 / 纸感浅色。
- 主色：深墨色、冷灰、低饱和蓝或青作为强调。
- 卡片：少量使用，边框轻、阴影轻。
- 页面：信息密度适中，像 Linear / Stripe Docs / Anthropic Docs 的克制风。

## 16. 内容原则

- 不写“AI 将彻底改变一切”这种空话。
- 每个概念都要回答“工程上怎么用”。
- 每道面试题都要回答“面试官想考什么”。
- 每个项目都要回答“怎么写进简历”。
- 内容引用外部材料时优先官方文档，避免搬运长文。

## 20. 成功标准

第一阶段成功：

- 访问者 3 分钟内知道 AI Agent 工程师要学什么。
- 访问者可以从首页进入路线、面试题和项目。
- 你可以用站点内容辅助准备至少 30 道面试题。
- 站点能被后续 AI agent 按文档继续开发。

第二阶段成功：

- 你能在简历中写出这个项目。
- 每个项目实战都有可展示代码或 Demo。
- 面试时能用站点内容组织回答。
